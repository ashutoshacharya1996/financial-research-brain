# 07_OPPORTUNITIES

## Purpose

This folder is the primary output of the Opportunity Discovery Engine.

An Opportunity is the system's central object: a convergence of Theme + Evidence + Candidate Stocks + Timing + Risk, specific and well-evidenced enough to direct research attention and downstream investment consideration.

---

## Folder Structure

```
07_OPPORTUNITIES/
├── README.md                        — this file
├── opportunity_schema.md            — canonical data model for all Opportunity records
├── weekly_output_template.md        — formatting contract for Sunday reports
├── weekly/                          — time-series of Sunday reports (one file per week)
│   └── YYYY-MM-DD.md
└── active/                          — live Opportunity records (one file per opportunity)
    └── <opportunity-id>.md
```

---

## Subfolders

### `weekly/`

Contains one file per Sunday, named `YYYY-MM-DD.md`.

Each file is generated by the Opportunity Screener agent and contains:
- **Section 1:** Top 10 Stocks for the Research Queue
- **Section 2:** Top 3 High-Conviction Opportunities
- **Signal Notes:** What changed vs. the prior week

Do not edit these files after they are generated. They are the historical record.

### `active/`

Contains one file per active Opportunity, named `<opportunity-id>.md`.

Opportunity files are created when a research queue signal graduates to conviction-stage (corroboration_count ≥ 3, ≥ 2 timing catalysts, ≥ 1 risk). They are updated weekly by the Opportunity Screener as new evidence arrives.

When an Opportunity's status changes to Closed, its file moves to `closed/` (a future folder to be created when needed).
